Select container type

The container type defines where the container is deployed. There are three types of containers:

  • Child container - a container that is created on the same machine as an existing root container. The child runs in its own VM, but the parent container can stop and start the container.

Important: Child containers cannot be parent containers.

Tip: Child containers should not be used for Fabric registry agents.

  • Remote root container - a container that is created independent of other containers. Root containers can be parents of child containers.
  • Cloud-managed Fuse container - a root container that is deployed on a host obtained from a cloud provider. Fuse Fabric works with most popular cloud providers including Amazon and RackSpace.

Note: You can use an internal cloud provider
