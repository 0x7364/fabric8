Creating a Container in the Cloud

Overview

Creating a container in the cloud installs a new Fuse container on a server instance in your cloud. Fuse Management Console will do the following:

  1. Connect to your cloud provider.

  2. Create a new server instance in the cloud.

  3. Download the binary files required to install a Fuse container.

  4. Install the container in a specified user's home folder.

  5. Start the container and provision all of the specified profiles.

Procedure

To create a root container in a cloud:

  1. Connect to your cloud provider by adding it to Fuse Management Console.

    See Adding a Cloud Provider.

  2. Select Containers from the main menu.

    The Containers page opens.

  3. Click Create Fuse Container.

    The Create Fuse Container page opens.

  4. In the Name field, enter a name for the new container.

    The name can only contain letters, numbers, and underscores(_).

  5. If you want to make the new container a registry server check Make container a distributed registry server.

    Tip

    It is recommended that if you create a registry server that you do not deploy any other services into the container.

  6. If you want to pass any arguments to the container's JVM, check the JVM Options field and enter the options.

  7. If you need to use a Maven proxy to access the repository containing your Fuse artifacts, check the Maven Proxy URL field and enter the URL for your Maven proxy.

  8. If you want to create multiple instances of the container enter a number greater than one in the Count field.

    When you create multiple containers each container's number is appended to the value specified in the Name field.

  9. Select how the container will report its IP address to other containers from the options in the Resolver drop-down list.

    The options are:

    Local Hostname

    When requests for the container's location are made, the container's local hostname is returned.

    Local IP

    When requests for the container's location are made, the container's local IP address is returned.

    Public Hostname

    When requests for the container's location are made, the container's public hostname address is returned.

    Public IP

    When a requests for the container's location are made, the container's public IP address is returned.

    Important

    Cloud containers that use Local IP or Local Hostmane will not be accessible to containers that do not reside in the same cloud provider.

    Important

    Amazon EC2 and Rackspace only support using public host names. Containers deployed to either service will always use Public Hostname regardless of the option chosen.

  10. Click Next.

    The Select profiles to install on the container page opens.

  11. Select the version of profiles you want to use for provisioning the container.

    The value of this drop down defaults to the value set on the Profiles page.

  12. Select one or more profiles to deploy to the container.

  13. Click Next.

    The Select the type of container page opens.

  14. Select Cloud-managed Fuse container.

  15. Click Next.

    The Cloud Service Configuration page opens.

  16. Select your cloud provider from the Cloud Provider drop-down list.

  17. Select a size for the new server instance from the Instance Type drop-down list.

  18. In the Instance Selection section, choose if you want to configure the new server instance by operating system or by image ID.

  19. Click Next.

    The second Cloud Service Configuration page opens.

  20. Configure the server instance that will be created.

    If you are configuring the server instance by operating system:

    1. In the OS Family field, select the operating system the server will use.

    2. In the OS Version field, select the version of the operating system the server will use.

    If you are configuring the server instance by ID and location:

    1. In the Image ID field, enter the ID of the image to use when instantiating the virtual machine.

    2. In the Location ID field, enter the ID of the location where the new server instance will be deployed.

  21. Click Next.

    The next Cloud Service Configuration page opens.

  22. In the User Account field, enter the name of the user to use when installing the new container on the new server instance.

    The specified user will be created on the new server instance and granted the required permissions to install the container.

  23. Enter an optional image owner in the Image Owner field.

  24. Enter an optional group tag in the Group Tag field.

  25. Click Finish.

    The wizard closes and the new container will appear in the list of containers. Initially the Active indicator may be red, but should turn green shortly after adding it. The Provisioned indicator may also show gray bars, but should turn green when the specified profiles are fully provisioned.
